Eventory: A Software Engineering Class Project

Overview

Role/Tools

I guided project development as second-in-charge and contributed to front end coding and creation of use case diagrams
Tools: Java, Android Studio, draw.io

Problem

When it comes to finding things to do, a main source of event publication is via Facebook. However, searching for events on Facebook isn't easy and you often can only see events that your mutual friends are already interested in. This leaves you with a limited amount of event exposure.

Solution

Our final solution is an Android Application called Eventory. We hoped that clubs and organizations on campus would cross post their events from Facebook to our application. Then, users could see events that many people were interested in and filter events by categories to find things that aligned with their own interest. This was our main functionality.We wanted users to be able to find events to go to in an easier manner than what current alternatives allowed. We also learned through a survey we put out that users are often spontaneous when it comes to going to an event. For this, we allowed our main page to be sorting events by time of creation, events going on right now and events with a lot of interest gathered.

Github Link

Process

We started off outlining all the user stories and use cases we would have. This allowed us to flesh out the features we wanted and to place them in a logically manner to be reached by users. This allowed us to set up our information architecture which allowed us to guide our development. We split into two teams, front end and back end development. We finished off by creating test cases and conducting internal testing.
