UX Portfolio: Behind the Scenes

Overview

Role/Tools

This has been a solo endeavor by myself to create my own portfolio
Tools:Google Analytics, User Testing, HTML/CSS, Javascript, MaterializeCSS Framework

Problem

All User Experience Designers need portfolios. After experimenting with website builders such as Wix and Wordpress, I decided to try and build my own since I did not want to be restricted by the design decisions of the premade templates. I started off by finding a template online and modifying the layout to fit my own preferences. My desire to improve my front-end development skills led me to decide to ultimately create one from scratch.

Solution

The portfolio you are currently viewing is created with HTML/CSS with a Materialize CSS Framework and Javascript.

Challenges

Among the challenges I faced is finding out what format would appeal best to my users and learning how to quantify my success with data. My data may be skewed by my own time spent on my website which I have now altered to exclude my IP address from future datasets. In addition, the time period and subsequent data for my current portfolio iteration is significantly decreased compared to my previous iteration, making it more difficult to see patterns.

Previous iterations of portfolio website

Process

First Iteration

My initial layout showcased my projects on my homepage with short blurbs introducing this project and had a project navigation system where you could either click back to home and find a specific project or traverse all my projects via next and previous buttons. The rationale behind this was so that recruiters looking through my website could see which projects piqued their interest. However, this also forced users to scroll down and skim every single project regardless of whether or not it was relevant. This is important because my projects cover a wide variety of disciplines.

Second Iteration

For my second iteration, I wanted to keep the idea of having project blurbs so that people visiting my website could see a description of each project before having to click through to a project. To view each project, I included navigation links in the footer as well as the option to click back to main projects page to see all project blurbs again. I also separated my projects into design and code but quickly realized I couldn’t fully separate the projects into the two disciplines. I tested this version with my peers and user feedback felt that the lack of images made my portfolio visually unappealing. I wanted to control the projects that users would see first and so decided to make my home page the project I felt showcased my skills best. Watching my testers use my site, they disregarded my home page project and clicked on the projects they wanted to see anyway. Since my Google Analytics data shows actual recruiters using my website, I heavily relied on it to make the current iteration of my website. At the time, I was only recording overall visits and visits to each page. I noticed a disproportionate number of users clicking onto the home page after a projects page. This led me to conclude that users weren’t using my navigation links in the footer since they were heading back to my home page to navigate my projects. From this, I went onto my third iteration.

Data from Google Analytics
Final Iteration

For my final iteration, I concluded that my portfolio has two key audiences: recruiters and designers.
Recruiters: Important to make a good first impression in order to be considered for a role. Might expect visual appeal because of designer roles
Designers: Need to showcase my work flow and show that I know what I am doing. Might be interested in the indepth process and research and also UI skills
I decided to examine other ux designer portfolios and imitate the same style with image based project buttons opening into project case studies and an initial home page with my image and a short blurb about myself. To allow users to know which projects are relevant to the job I am applying to, they can sort by tags to see which projects I like the most as well as which ones might show the skills they are looking for.
